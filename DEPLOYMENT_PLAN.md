## 二、方案对比（3个推荐方案）

### ┌─────────────────────────────────────┐
### │ 方案A：国内云服务器（推荐）          │
### └─────────────────────────────────────┘

**架构**：阿里云/腾讯云免费试用服务器 → Flask系统 → 阿里云盘/OneDrive同步

| 项目 | 说明 |
|------|------|
| 服务器 | 阿里云"飞天免费计划"(新用户12个月免费) 或 腾讯云轻量(约35元/月) |
| 系统 | 当前Flask系统原封不动部署，SQLite照用 |
| 数据同步 | 阿里云盘API 或 OneDrive(世纪互联版) + rclone定时同步 |
| 表单录入 | 系统自带响应式界面，手机直接用浏览器填 |
| 优点 | 国内访问快、完全掌控、数据在国内、原系统不需改造 |
| 缺点 | 阿里云12个月后需续费(~35元/月)、需身份证实名认证 |
| 适合 | 长期使用、对数据安全有要求 |

**费用**：前12个月免费 → 之后约35元/月

---

### ┌─────────────────────────────────────┐
### │ 方案B：Vercel + Supabase（免费长期） │
### └─────────────────────────────────────┘

**架构**：Vercel(前端+API) → Supabase(PostgreSQL免费数据库) → 系统改造

| 项目 | 说明 |
|------|------|
| 服务器 | Vercel免费托管，国内大部分地区可访问 |
| 数据库 | Supabase免费PostgreSQL(500MB)，国内可访问 |
| 数据同步 | Supabase自带导出 + 定时备份到阿里云盘 |
| 表单录入 | 系统自带界面 |
| 优点 | 真正长期免费、全球CDN、自动HTTPS |
| 缺点 | 需改造现有系统(Flask→FastAPI/API、SQLite→PostgreSQL)、偶有冷启动延迟、Vercel国内偶尔慢 |
| 适合 | 预算为零、愿意折腾技术、能接受偶尔慢一点 |

**费用**：完全免费（Supabase免费500MB足够）

---

### ┌─────────────────────────────────────┐
### │ 方案C：Termux本机 + 内网穿透         │
### └─────────────────────────────────────┘

**架构**：你的手机(Termux)跑Flask → frp/ngrok内网穿透 → 外网可访问

| 项目 | 说明 |
|------|------|
| 服务器 | 你的Android手机上跑Termux(就是现在在用的) |
| 穿透 | frp到免费frp服务器 或 cloudflared tunnel |
| 数据同步 | rclone定时同步SQLite文件到阿里云盘/OneDrive |
| 表单录入 | 系统自带界面，穿透后任何设备可访问 |
| 优点 | 零成本、数据完全在手里、无需注册任何平台 |
| 缺点 | 手机得开着Termux、穿透服务可能不稳定、手机耗电、性能有限 |
| 适合 | 试玩/过渡阶段、不想注册任何云平台 |

**费用**：完全免费（frp公共服务器可能有限速）

## 四、数据同步方案详细设计

### ⚠️ 关于Google Drive的重要说明

Google Drive API 在国内被完全封锁，无法直接使用。替代方案：

| 方案 | 可行性 | 说明 |
|------|--------|------|
| Google Drive + VPN | ⚠️ 勉强可用 | 需要每次开VPN，Token刷新也需要Google访问，不适合自动化 |
| OneDrive(世纪互联版) | ✅ 推荐 | 微软与世纪互联合作的国内版，rclone原生支持 |
| 阿里云盘 | ✅ 推荐 | 国内原生，有非官方API(aliyundrive-webdav) |
| 百度网盘 | ⚠️ 一般 | API限制多、限速，不推荐 |

### 推荐同步方案：rclone + OneDrive/阿里云盘

```
每天定时执行：
1. SQLite数据库文件 → rclone copy → 云盘/vehicle-backup/
2. 文件名带日期：vehicle_2026-04-30.db
3. 保留最近30天备份，自动清理旧文件
```

**配置命令**（以OneDrive为例）：
```bash
# 安装rclone
pip install rclone-python  # 或直接下载rclone二进制

# 配置OneDrive(世纪互联版)
rclone config
# 选择 onedrive → 世纪互联版 → 按提示完成OAuth

# 手动同步测试
rclone copy ~/vehicle-management/vehicle.db onedrive:vehicle-backup/

# 设置定时同步（cron）
0 18 * * * rclone copy ~/vehicle-management/vehicle.db onedrive:vehicle-backup/vehicle_$(date +\%Y\%m\%d).db
```

## 五、手机表单录入优化方案

当前系统已经有响应式界面，但手机上填表格单体验可以进一步优化：

### 优化1：添加"快速录入"模式

在手机端提供精简表单，只填必填项：
- 使用登记：日期 + 车辆 + 驾驶人 + 事由 → 里程事后补
- 费用登记：日期 + 车辆 + 类型 + 金额 → 详情事后补

### 优化2：微信/钉钉消息提醒

通过cron定时任务，每天下午5点推送：
- 今日未录入使用的车辆提醒
- 本周费用统计摘要
- 维保到期提醒

### 优化3：PWA支持（离线可用）

将系统改造为PWA(Progressive Web App)：
- 添加到手机主屏幕，像App一样打开
- 支持离线填写，有网时自动同步
- 推送通知

## 六、系统安全加固

作为政府办公系统，需要注意：

| 措施 | 说明 |
|------|------|
| 登录验证 | 添加简单密码保护（Flask-Login或HTTP Basic Auth） |
| HTTPS | Vercel/阿里云自动提供；自部署用Let's Encrypt |
| 数据备份 | 每日自动备份到云盘 + 每周手动导出CSV存档 |
| 访问限制 | 部署后限制只允许办公网IP访问（可选） |
| 操作日志 | 记录谁在什么时间添加/修改了什么记录 |
