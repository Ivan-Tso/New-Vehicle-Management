# AI 保养分析功能配置指南

## 1. 功能说明

AI 保养分析模块现已升级为真正的 RAG (检索增强生成) 系统，具备以下特性：

- **真实 AI 分析**：调用 GLM-4.5/DeepSeek 等大语言模型生成建议
- **手册解析**：支持上传 PDF/TXT 格式的车辆手册，自动提取文本
- **多文件支持**：可同时上传多个手册文件
- **历史记录整合**：结合过往维修记录生成个性化建议
- **个性化建议**：基于车辆里程、车龄、手册建议和历史记录

## 2. 配置 API Key

### 方法 1: 环境变量（推荐）

在服务器上设置环境变量：

```bash
export ZHIPU_API_KEY="your_zhipu_api_key_here"
```

### 方法 2: .env 文件

1. 复制示例文件：
   ```bash
   cp .env.example .env
   ```

2. 编辑 `.env` 文件，填入你的 API Key：
   ```
   ZHIPU_API_KEY=your_actual_key_here
   ```

### 获取 Zhipu API Key

1. 访问 https://open.bigmodel.cn/
2. 注册/登录账号
3. 进入控制台创建 API Key
4. 复制 Key 到配置文件中

## 3. 使用流程

### 3.1 上传手册

1. 进入 "AI 分析" 页面
2. 选择车辆
3. 点击文件上传框，可选择多个 PDF/TXT 文件
4. 点击 "🤖 分析" 按钮

### 3.2 查看分析结果

系统会：
1. 自动提取上传手册中的文本
2. 检索该车辆的历史维修记录
3. 调用 AI 生成个性化建议
4. 在页面显示分析结果

### 3.3 分析内容

AI 会提供：
- 立即需要进行的保养项目
- 未来 3 个月内到期的项目
- 长期保养规划建议
- 基于手册的特殊注意事项

## 4. 技术细节

### 支持的文件格式

- PDF（电子版手册，可提取文本）
- TXT（纯文本格式）

### 数据存储

- 上传的手册会提取文本后存储在 `uploaded_manuals` 表的 `extracted_text` 字段
- 单次提取文本限制为 100,000 字符（防止过大）
- 历史维修记录查询最近 5 条

### API 调用

- 默认模型：GLM-4.5 (via Zhipu API)
- 备用模型：DeepSeek-Chat
- 超时设置：30 秒
- 温度参数：0.3（保证建议一致性）

## 5. 故障排除

### 问题：提示 "API key not configured"

**解决方案**：检查是否正确设置了 `ZHIPU_API_KEY` 环境变量或 `.env` 文件。

### 问题：PDF 上传后无法提取文本

**可能原因**：
- PDF 是扫描版（图片格式），pypdf 无法提取
- PDF 有加密保护

**解决方案**：
- 使用文字版 PDF（可从官网下载）
- 或手动输入手册关键内容到 TXT 文件

### 问题：AI 响应超时

**可能原因**：网络问题或 API 服务不可用

**解决方案**：
- 检查网络连接
- 稍后重试
- 考虑使用国内 API 提供商

## 6. 费用说明

- Zhipu API 按 token 计费
- 单次分析约 1000-2000 tokens
- 具体价格参考 https://open.bigmodel.cn/pricing

## 7. 隐私与安全

- 上传的手册文件存储在服务器本地
- 文本内容会发送到 AI 服务商进行处理
- 建议不要上传包含敏感信息的手册
